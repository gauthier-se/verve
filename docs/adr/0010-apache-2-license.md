# Apache-2.0 license

## Context

Verve is open source. The license is hard to change once external contributors
hold copyright, so it is chosen up front. The main axis is permissive
(MIT/Apache) vs copyleft (GPL/AGPL, the latter closing the SaaS loophole).

## Decision

License under **Apache-2.0**.

## Why

Chosen deliberately over the recommended AGPL-3.0. Apache-2.0 maximizes adoption
(no AGPL bans, usable in commercial and closed contexts), includes an explicit
patent grant, and is a well-understood permissive standard. The trade-off
accepted: anyone may run Verve as a closed SaaS or ship proprietary
forks/connectors without contributing back — deemed acceptable in favor of the
widest possible reach and contribution surface.
